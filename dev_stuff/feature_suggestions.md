# Feature Suggestions for Danktunes

## Playback Features
- [ ] Queue system (separate from playlist)
- [ ] Smart shuffle (avoid recent repeats)
- [ ] Gapless playback
- [ ] Sleep timer
- [ ] Last position memory (podcasts/audiobooks)
- [ ] Crossfade between tracks
- [ ] ReplayGain support

## UI/Visual Features
- [x] Audio visualizer (spectrum analyzer)
- [x] Album art display (detection + iTerm2/Kitty/Sixel/urxvt/Konsole protocols)
- [ ] Lyrics display
- [ ] Progress bar with seek preview
- [ ] Mini player mode (compact)
- [ ] Theme support (dark/light/custom)

## Library Management
- [ ] Sort by: name, date, duration, play count
- [ ] Favorites/bookmarks
- [ ] Auto-playlists (most played, recently added)
- [ ] Tag editing (artist, title, album)
- [ ] Duplicate detection

## Smart Features
- [ ] Auto-dj (smart random based on genre/artist)
- [ ] Playlist generation from folder
- [ ] Queue from search results
- [ ] Batch file operations

## System Integration
- [ ] MPRIS support (Linux media controls)
- [ ] Global media keys
- [ ] Desktop notifications
- [ ] CLI mode for scripting
- [ ] PID file for single instance

## Advanced
- [ ] Plugin system
- [ ] Scriptable keybindings
- [ ] Audio format conversion
- [ ] Streaming support (internet radio)
- [ ] Multi-room sync

---

## Priority Recommendations

### High Priority (Quick Wins)
1. Queue system - Simple addition, high value
2. Smart shuffle - Uses existing shuffle history
3. Last position memory - Already partially implemented in Phase 4
4. Favorites/bookmarks - Already added in Phase 5

### Medium Priority
1. Audio visualizer - Visual appeal
2. Mini player mode - Useful for small terminals
3. Sleep timer - Popular feature
4. Sort options - Already added in Phase 5

### Lower Priority
1. Album art - Requires additional dependencies
2. Lyrics - Complex API integration
3. MPRIS - Linux-specific
4. Plugin system - Large undertaking
